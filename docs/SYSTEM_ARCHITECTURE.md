# AVATAR v1.1: System Architecture & Technical Manual

## 1. Концепция Системы
AVATAR — это платформа глубокой психологической диагностики и трансформации, работающая через нарративное погружение и анализ поведенческих паттернов. Версия 1.1 полностью переведена на **текстовую архитектуру** (Text-Based Diagnostics) с сохранением прецизионного сбора данных о поведении.

---

## 2. Жизненный Цикл Пользователя (User Journey)

Система ведет пользователя через непрерывный цикл развития из четырех этапов:

1.  **Onboarding (Вход)**: Сбор базовых целей и текущего состояния. Формирование первичного «намерения».
2.  **Synchronization (Диагностика)**: 5-слойное погружение в текстовую сцену. Цель — извлечение проекционного материала подсознания.
3.  **Alignment (Выравнивание)**: Работа с выявленным теневым паттерном, замена ограничивающего убеждения на новое через протоколы трансформации.
4.  **Integration (Проявление)**: Фиксация результатов, запись в Дневник и отражение изменений в Портрете (UserPortrait).

---

## 3. Экосистема Агентов (Agent Core)

### 🤖 MasterAgent (`master_agent.py`)
*   **Роль**: Фасад и координатор.
*   **Функция**: Переключает контексты между диагностикой и выравниванием, управляет доступом к специализированным агентам.

### 🔍 SyncAgent (`sync_agent.py`) — «Narrative Scanner»
*   **Роль**: Сборщик проективных данных.
*   **Механика**: Ведет пользователя через 5 слоев нарратива (Поверхность -> Символ -> Разлом -> Дно -> Ядро).
*   **Особенности**: Использует библиотеку готовых сцен (`TextScene`) и динамически адаптирует вопросы на основе телесного отклика пользователя.
*   **Контекст**: При каждом запуске запрашивает исторические данные из `UserPortrait` (прошлые символы и паттерны) для обеспечения непрерывности опыта.

### 🧠 AnalyticAgent (`analytic_agent.py`) — «The Mirror»
*   **Роль**: Интерпретатор и архитектор данных.
*   **Задачи**:
    *   **Phase Analysis**: Извлекает структурированные признаки (действия, эмоции, паттерны) из ответов.
    *   **Mirror Analysis**: Проводит финальный аудит сессии, определяет уровень Хокинса и ядро убеждения.
    *   **Portrait Update**: Агрегирует результаты сессии в `UserPortrait`, обновляя «карту сознания» пользователя.

### 🔄 EvolutionAgent (`evolution_agent.py`) — «Self-Learning Loop»
*   **Роль**: Оптимизатор системы.
*   **Задачи**: Анализирует эффективность текстовых сцен. Если сцена дает слабый диагностический отклик (короткие ответы, быстрый проход), агент помечает её как неактивную и генерирует замену через LLM.

### 📊 HawkinsAgent (`hawkins_agent.py`)
*   **Роль**: Метрический контроль.
*   **Функция**: Точная оценка вибрационного состояния по шкале уровней сознания Дэвида Хокинса.

## 4. Архитектура Данных (Data Layers)

Данные собираются на четырех уровнях глубины:

### Уровень 1: Сырая Телеметрия (`events`)
*   Таблица: `Event`
*   Данные: клики, паузы, прокрутки, время начала/конца ввода.

### Уровень 2: Поведенческие Признаки (`session_features`)
*   Таблица: `SessionFeatures` / `UserBehaviorProfileV2`
*   Метрики:
    *   `semantic_shift`: Дистанция между вектором входа (Слой 1) и вектором ядра (Слой 5).
    *   `narrative_depth`: Плотность и сложность ответов.
    *   `emotional_volatility`: Вариативность эмоциональных реакций.
    *   `avg_decision_speed`: Скорость реакции на «Разлом» (Слой 3).

### Уровень 3: Диагностические Ячейки (`sync_sessions`)
*   Таблица: `SyncSession`
*   Данные: `core_pattern` (сценарный паттерн), `recurring_symbol` (ключевой символ), `body_anchor` (телесный зажим/резонанс), `hawkins_score`.

### Уровень 4: Холистический Портрет (`user_portrait`)
*   Таблицы: `UserPortrait`, `Pattern`, `Connection`
*   **Суть**: Это «Единый Источник Истины». Здесь хранятся агрегированные данные по каждой Сфере жизни (Деньги, Отношения, Тело и т.д.) и сквозные связи между ними.

---

## 5. Петля Самообучения (The Feedback Loop)

1.  **Взаимодействие**: Пользователь проходит сессию, данные падают в `SceneInteraction`.
2.  **Анализ**: `FeatureExtractor` превращает текст и время в векторные метрики.
3.  **Оценка**: `EvolutionAgent` замеряет «Диагностическую мощь» сцены (Diagnostic Power Score).
4.  **Эволюция**: Сцены с низким скором заменяются новыми, более эффективными для конкретного архетипа/сферы.

---

## 6. Инструкции для Разработчиков

*   **Добавление новых данных**: Если вы добавляете новый параметр сбора, обязательно обновите `update_user_portrait` в `AnalyticAgent`, чтобы данные попали в долгосрочную память системы.
*   **Работа с LLM**: Все промпты находятся внутри функций `build_avatar_prompt` и аналитических функций. Используйте `temperature=0.2` для анализа и `0.8+` для генерации нарратива.
*   **Миграции**: При изменении моделей данных используйте скрипты в `/tmp` для предварительной проверки на живых данных перед фиксацией схемы.
