# AVATAR: Полное Руководство по Архитектуре и Методологии (v1.2)

## 1. Философия и Миссия Системы
AVATAR — это не просто приложение, а «Платформа Эволюции Сознания». В её основе лежит синтез древних систем (астрология, юнгианская психология, таро) и современных технологий (LLM, мультиагентные системы). 

**Цель**: Перевести абстрактные знания о личности в осязаемую цифровую модель (Аватар), с которой пользователь может взаимодействовать для роста, выравнивания внутренних конфликтов и реализации потенциала.

---

## 2. Путь Героя: Жизненный Цикл Пользователя

### Этап 1: Протокол Синхронизации (Onboarding)
Система собирает прецизионные данные для построения «Ядра»:
- **Натальные данные**: Дата, время и место рождения (проходят верификацию через геокодинг).
- **Персонализация**: Пол пользователя и язык общения (на текущий момент — Русский).
- **Расчет**: На основе эфемерид строится натальная карта, которая мапится на 12 Сфер и 12 Архетипов.

### Этап 2: Визуальный Резонанс
Пользователь проходит серию интуитивных выборов изображений. Это позволяет системе замерить текущее состояние «в моменте», которое может отличаться от натального потенциала.

### Этап 3: Повседневная Эволюция
- **Daily Sync**: Прохождение текстовых сценариев, генерируемых `SyncAgent`.
- **Reflection**: Диалоги с ИИ-психологом (`ReflectAgent`) по методологии «3 Зеркал».
- **Master Hub**: Центр управления 12-ю сферами жизни, где пользователь видит прогресс в каждой области.

---

## 3. Методологический Фундамент

### 12 Сфер Жизни (Spheres)
Жизнь пользователя разделена на 12 секторов (аналог астрологических домов):
1. **IDENTITY** (Личность) — Эго и самовыражение.
2. **RESOURCES** (Ресурсы) — Деньги и самоценность.
3. **COMMUNICATION** (Связи) — Окружение и информация.
... и так далее до 12. **SPIRIT** (Дух) — Завершение и тишина.

### 22 Архетипа (Archetypes)
Мы используем систему 22 Старших Арканов как универсальные психологические паттерны:
- Каждый Архетип имеет **Свет** (ресурсное состояние) и **Тень** (ограничивающее убеждение).
- Связь планета/знак -> Архетип позволяет строить мост между астрологией и психологией.

### Шкала Хокинса (Vibration Levels)
Каждая сессия и состояние оценивается по шкале уровней сознания (от 0 до 1000). Это дает количественную метрику эволюции пользователя.

## 4. Гексагональная Архитектура: Cortex (AI Multi-Agent System)

Система управляется сетью специализированных агентов, каждый из которых имеет свою «личность» и задачу.

### 🤖 MasterAgent
**Функция**: Оркестратор. Решает, какой агент должен вступить в игру в зависимости от запроса пользователя. Хранит глобальный контекст.

### 🔍 SyncAgent (The Narrative Engine)
**Функция**: Создает «игровые» сценарии для диагностики.
- Использует 5 слоев погружения: *Поверхность -> Символ -> Разлом -> Дно -> Ядро*.
- Генерирует вопросы, учитывая пол и язык пользователя.

### 🧠 ReflectAgent (The AI Psychologist)
**Функция**: Проводит глубокие терапевтические сессии.
- Работает по системе «3 Зеркал»: *Зеркало Фактов -> Зеркало Тени -> Зеркало Потенциала*.
- Помогает пользователю осознать теневой паттерн и трансформировать его.

### 📊 AnalyticAgent
**Функция**: «Сшивает» все данные. Извлекает из текста паттерны, символы и вычисляет Hawkins Score. Формирует «Hero Essence» — краткую выжимку сути пользователя.

### 🔄 EvolutionAgent
**Функция**: Контент-менеджер системы. Анализирует, какие сценарии работают лучше, и переписывает неэффективные описания архетипов.

## 5. Технический Стек

### Backend (Python/FastAPI)
- **Engine**: Асинхронная архитектура, интеграция с OpenAI API.
- **DB**: PostgreSQL + SQLAlchemy. Хранение натальных карт, чатов и векторов.
- **Calculations**: Собственные модули для работы с астрологическими эфемеридами.

### Frontend (React/Next.js)
- **UI/UX**: Стиль «Sacred Geometry». Использование Glassmorphism, сложных градиентов и кастомных Canvas-графиков.
- **Animations**: Framer Motion для плавных переходов между состояниями сознания.
- **State**: Zustand для синхронизации данных пользователя на лету.

---

## 6. База Данных (Основные Сущности)
- `User`: Базовая инфо, `gender`, `language`, `natal_data`.
- `CardProgress`: Статус развития каждого из 22 архетипов в каждой из 12 сфер.
- `SyncSession`: Хранилище пройденных нарративных путей.
- `AIDiagnosticSession`: Лог диалога при первом входе.

---

## 7. Безопасность и Этическая Модель
- Полная анонимность: данные используются только для уточнения AI-подсказок.
- Запрет на медицинские диагнозы: ИИ выступает как коуч и зеркало, а не как врач.

## 8. Структура Проекта (Core Map)

Для быстрой навигации по системе:

- **/backend/app/agents/**: «Мозговой центр». Здесь лежат промпты и логика всех ИИ-агентов (`sync`, `reflect`, `master` и т.д.).
- **/backend/app/routers/**: API эндпоинты. Логика расчетов натальных карт — `calc.py`, работа с чатом — `reflect.py`.
- **/backend/app/models/**: Схема базы данных (SQLAlchemy). Отражение всей структуры Аватара в цифре.
- **/backend/data/**: Знания системы. JSON-матрицы сфер, архетипов, шкал Хокинса и наборы текстовых сцен.
- **/frontend/src/app/**: Визуальный слой (Next.js). Страницы онбординга, мастер-хаба и карточек.
- **/frontend/src/lib/api.ts**: Интерфейс взаимодействия фронтенда с бэкендом.

## 9. Философия Разработки (AGILE & AI-FIRST)
1. **Тень — это Ресурс**: Мы никогда не говорим пользователю, что он «плохой». Мы показываем, где заперта его энергия.
2. **Контекст превыше всего**: Каждый ответ ИИ должен строиться на базе `gender`, `language` и истории прошлых сессий.
3. **Чистота Архитектуры**: Расчеты отделены от интерпретаций. Данные отделены от промптов.
