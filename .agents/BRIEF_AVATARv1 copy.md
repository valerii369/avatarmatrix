# ЗАДАЧА ДЛЯ DEV-АГЕНТА — AVATARv1.0
## Онбординг + Векторная база карточек

> Проект: AVATARv1.0
> Версия: 1.1
> Синхронизация: JSON → Supabase pgvector

---

## КОНТЕКСТ СИСТЕМЫ

AVATARv1.0 — психологическая трансформационная система в Telegram.
Пользователь проходит онбординг → агент диагностирует состояние через разговор →
проявляет карточки архетипов по сферам жизни → пользователь растёт через карточки.

Карточки основаны на матрице: 22 архетипа × N сфер × 10 уровней Хокинса.

---

## ЗАДАЧА 1 — ОНБОРДИНГ: ДВА ПУТИ ВХОДА

Первый экран после /start — выбор из двух кнопок:

Кнопка 1: "Доверяю себе"
Кнопка 2: "Доверяю астрологии"

### ПУТЬ 1 — "Доверяю себе" (LIVE SCAN)

Пользователь нажимает "Доверяю себе"
        ↓
Агент ведёт диалог — 7 вопросов поочерёдно:
  1. "Опиши своё состояние прямо сейчас — одним словом или предложением"
  2. "Что сейчас занимает больше всего мыслей?"
  3. "Есть что-то что хотел изменить давно — но не сдвигается?"
  4. "Что уже пробовал с этим делать?"
  5. "Как выглядела бы твоя жизнь через год — честно?"
  6. "Когда что-то идёт не так — ты злишься, уходишь в себя или контролируешь?"
        ↓
Агент накапливает ответы
        ↓
Отправляет текст в vector similarity search
        ↓
Получает ranked list карточек
        ↓
Проявляет карточки по приоритету

Профиль после LIVE SCAN сохраняет: идентификатор пользователя, тип онбординга,
сырые ответы из диалога, определённый архетип, активные сферы в порядке приоритета,
уровень Хокинса, список проявленных карточек со score, накопленный опыт и уровень.

### ПУТЬ 2 — "Доверяю астрологии" (NATAL MAP)

Пользователь нажимает "Доверяю астрологии"
        ↓
Агент запрашивает три параметра:
  1. Дата рождения
  2. Время рождения (если не знает — примерно или пропустить)
  3. Город рождения
        ↓
Система передаёт данные в астрологический калькулятор
        ↓
Получает доминантный архетип и активные сферы
        ↓
Использует как фильтр для vector search
        ↓
Проявляет карточки пользователю

Профиль после NATAL MAP сохраняет всё то же что LIVE SCAN,
плюс дату, время и город рождения.

## ЗАДАЧА 2 — СТРУКТУРА КАРТОЧКИ (ЭТАЛОН)

Каждая карточка делится на два блока: vector и not_vector.

### Блок VECTOR — идёт в векторную базу

Все поля конкатенируются и векторизуются через OpenAI Embeddings.
По ним агент ищет совпадение с речью пользователя через similarity search.

Теневое проявление
  — деструктивные паттерны архетипа в данной сфере

Световое проявление
  — здоровые паттерны архетипа в данной сфере

Общее описание
  — суть отношений архетипа с данной сферой

10 уровней Хокинса — как выглядит состояние на каждом уровне
в данной сфере для данного архетипа:

  Уровень 1:  0–20     Небытие / Стыд
  Уровень 2:  20–100   Горе / Вина
  Уровень 3:  100–175  Страх / Тревога
  Уровень 4:  175–200  Гнев / Гордыня
  Уровень 5:  200–250  Мужество
  Уровень 6:  250–310  Нейтральность
  Уровень 7:  310–400  Готовность
  Уровень 8:  400–500  Разум
  Уровень 9:  500–600  Любовь / Радость
  Уровень 10: 600–1000 Мир / Просветление

Маркеры речи теневого состояния
  — конкретные фразы диагностики тени

Маркеры речи светлого состояния
  — конкретные фразы диагностики света

### Блок NOT_VECTOR — метаданные логики

Не векторизуются. Управляют поведением системы после нахождения карточки.

Идентификатор карточки
  — уникальный код. Формат: СФЕРА_НОМЕРАРХЕТИПА

Номер архетипа — от 0 до 21
Название архетипа — текстовое название
Сфера — к какой сфере жизни относится

Диапазоны Хокинса
  — три метки (низкий / средний / высокий) для фильтрации при поиске

Сигналы приоритета
  — по каким сигналам речи карточка становится первоочередной
    среди всех найденных

Вес карточки
  — числовой множитель важности. Влияет на итоговый score при ранжировании

Минимальный порог
  — ниже этого similarity score карточка не проявляется

Квантовый сдвиг
  — конкретное действие или осознание. Следующий практический шаг пользователя.

Вопрос агента
  — вопрос для активации и углубления работы с карточкой

Связанные карточки
  — идентификаторы карточек которые открываются следующими. Цепочка роста.

Уровень прогрессии — к какому уровню системы относится карточка
Награда — опыт за проработку карточки

## ЗАДАЧА 3 — ЛОГИКА ПРОЯВЛЕНИЯ КАРТОЧЕК

### Scoring

similarity_score из pgvector (0.0–1.0)
  × вес карточки из not_vector
  × повышающий коэффициент при совпадении сигналов приоритета
  = итоговый ranked_score

### Пороги (тестовый режим)

ranked_score > 0.7   → проявить карточку активно
ranked_score 0.5–0.7 → упомянуть как связанное
ranked_score < 0.5   → не показывать

В тестовом режиме — проявлять все карточки выше 0.5 без ограничения по количеству.

### Порядок показа

Агент показывает карточки по убыванию ranked_score.
Для каждой карточки:
  1. Отражает что услышал в словах пользователя
  2. Задаёт вопрос активации из карточки
  3. После ответа — показывает квантовый сдвиг
  4. Начисляет награду
  5. Предлагает связанные карточки как следующий шаг

## ЗАДАЧА 4 — СИНХРОНИЗАЦИЯ JSON → SUPABASE

JSON файл карточки создан или обновлён
        ↓
Скрипт синхронизации
        ↓
Читает блок vector → конкатенирует все поля → векторизует (OpenAI text-embedding-3-small)
        ↓
Сохраняет векторное представление в Supabase pgvector
        ↓
Сохраняет метаданные not_vector в ту же запись

Формат имени JSON файла: archetype_{номер}_{название}.json
Пример: archetype_0_fool.json

Синхронизация запускается при:
- добавлении новых карточек
- обновлении текстов существующих карточек
- изменении маркеров речи или сигналов приоритета

---

## ЗАДАЧА 5 — ТЕСТОВЫЙ РЕЖИМ

При включённом тестовом режиме агент дополнительно показывает:
- какие маркеры речи сработали для каждой карточки
- score каждой найденной карточки
- почему выбрал именно эту сферу как первоочередную

1. Настроить Supabase — таблицы карточек и профилей, расширение pgvector

3. Реализовать онбординг — экран выбора + два пути диалога
4. Реализовать диагностический агент со scoring логикой
5. Реализовать проявление карточек
6. Включить тестовый режим с логированием scores
7. Проверить на реальном диалоге
