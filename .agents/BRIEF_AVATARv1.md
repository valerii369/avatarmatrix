# БРИФ — AVATARv1.0 / Онбординг + Векторная база

---

## ОБЩАЯ ИДЕЯ

Пользователь заходит в Telegram-бота и проходит онбординг через живой разговор с агентом.
Агент анализирует речь, определяет состояние пользователя и проявляет карточки архетипов.
Карточки хранятся в векторной базе Supabase и синхронизируются через JSON файлы.

---

## ОНБОРДИНГ — ДВА ВХОДА

На старте пользователь видит выбор из двух кнопок:

**"Доверяю себе" — LIVE SCAN**
Агент задаёт 7 вопросов в живом диалоге, анализирует ответы,
определяет активные сферы и архетип через similarity search по векторной базе.

**"Доверяю астрологии" — NATAL MAP**
Агент запрашивает дату, время и город рождения,
определяет архетип и активные сферы через астрологический расчёт,
затем также уходит в векторную базу за карточками.

Оба пути приводят к одному результату — профилю пользователя и набору проявленных карточек.

---

## КАРТОЧКИ — СТРУКТУРА ДАННЫХ

Каждая карточка делится на два блока.

### В векторную базу — similarity search:
- теневое проявление архетипа в сфере
- световое проявление архетипа в сфере
- общее описание архетипа в сфере
- проявление на каждом из 10 уровней шкалы Хокинса:
  0–20, 20–100, 100–175, 175–200, 200–250, 250–310, 310–400, 400–500, 500–600, 600–1000
- фразы-маркеры теневого состояния
- фразы-маркеры светлого состояния

### Не в вектор — метаданные логики:
- идентификатор карточки
- номер и название архетипа
- сфера жизни
- диапазоны уровней Хокинса (низкий / средний / высокий) для фильтрации
- сигналы приоритета — по каким маркерам речи карточка становится первоочередной
- вес карточки — множитель для итогового score
- минимальный порог score для проявления
- квантовый сдвиг — следующий шаг для пользователя
- вопрос агента для активации карточки
- связанные карточки — цепочка роста
- уровень прогрессии и награда за проработку

---

## ЛОГИКА ПРОЯВЛЕНИЯ

Агент получает текст пользователя → ищет по векторной базе →
получает ranked list карточек → проявляет все карточки выше порога score →
показывает в порядке приоритета.

На тестовом периоде агент проявляет все карточки которые считает релевантными,
без ограничения по количеству.

---

## СИНХРОНИЗАЦИЯ

Карточки ведутся в JSON файлах.
Скрипт синхронизации читает JSON → векторизует блок vector →
сохраняет векторное представление и метаданные not_vector в Supabase.
